# Arch Linux Dotfiles

This repository tracks the operational environment and user configurations for an owner-grade Arch Linux machine. It utilizes a **Bare Git Repository** structure. This method avoids the friction of symlinking configurations across the filesystem by mapping the `$HOME` directory directly as the Git working tree.

## Architecture Context

- **OS:** Arch Linux
- **Environment:** KDE Plasma 6 (Wayland)
- **Shell:** Zsh + Starship
- **Tracking Method:** Bare Git Repository (`~/.dotfiles.git`)
- **Execution Macro:** `dots`

---

## Deployment: Replicating on a New Machine

To pull these configurations onto a fresh Arch Linux installation, authenticate your SSH keys with GitHub, then execute this exact sequence.

### 1. Prevent Recursive Conflicts

Ensure the bare repository directory is ignored by the system to prevent Git from attempting to track its own database:

```bash
echo ".dotfiles.git" >> ~/.gitignore
```

### 2. Clone the Bare Repository

Clone the repository data from the remote endpoint directly into a hidden database folder. Do not use a standard `git clone` command.

```bash
git clone --bare git@github.com:iashutoshtiwari/dotfiles.git $HOME/.dotfiles.git
```

### 3. Establish the Execution Macro

Create a temporary alias in your active shell to route Git commands to the bare database:

```bash
alias dots='/usr/bin/git --git-dir=$HOME/.dotfiles.git/ --work-tree=$HOME'
```

### 4. Extract Configurations

Check out the tracked files directly into your root `$HOME` directory:

```bash
dots checkout
```

> **Note:** If checkout fails with an error regarding existing files (for example, a stock `.bashrc` or `.zshrc` generated by the OS), delete or move those default files and re-run `dots checkout`.

### 5. Suppress System Noise

Configure the local repository instance to ignore untracked files by default, preventing Git from displaying every cache file and document in your home folder:

```bash
dots config --local status.showUntrackedFiles no
```

## Daily Operations Workflow

Interact with this repository exactly like standard Git, substituting `git` with the `dots` alias (which is persistently sourced in `.zshrc`).

```bash
# Audit modified configurations
dots status

# Track a newly modified configuration file
dots add ~/.config/kwinrc

# Commit changes to the local history
dots commit -m "style: optimize kwin compositor window rules"

# Push to the remote endpoint
dots push
```

## System Package Synchronization

While this repository tracks how applications are configured, the package manifests dictate what is installed. To fully replicate this machine's capabilities on new hardware, install the dependencies stored in the manifest files.

### Restoring Packages

```bash
# 1. Install native repository packages
sudo pacman -S --needed - < pkglist.txt

# 2. Install AUR packages (requires yay)
yay -S --needed - < aurlist.txt
```

### Updating Manifests

Before pushing major system changes to this repository, regenerate the manifests to capture newly installed tooling:

```bash
pacman -Qeq > pkglist.txt
yay -Qeqv > aurlist.txt

dots add pkglist.txt aurlist.txt
dots commit -m "build: update system package manifests"
```
